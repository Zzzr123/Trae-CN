# 文档质量检查清单

## 内容完整性
- [ ] 是否覆盖了`structure.md`中定义的所有章节和关键元素？
- [ ] 各章节篇幅是否在预期范围内？
- [ ] 核心主题陈述（见`topic.md`）是否得到了充分论证？
- [ ] 是否引用了至少两处《麦田里的守望者》原文内容作为分析依据？
- [ ] 是否融入了至少3处个人真实经历或思考？

## 主题遵循度
- [ ] 是否出现了`topic.md`中"需要排除的内容"？
- [ ] 关键词是否被恰当使用（而非生硬堆砌）？
- [ ] 所有论述是否服务于核心主题，无明显偏题段落？
- [ ] 是否体现了软件工程专业与人文关怀的结合？

## 风格一致性
- [ ] 人称和视角是否全文统一，未出现混用？
- [ ] 总体基调和正式程度是否符合`voice.md`的设定？
- [ ] 是否存在`voice.md`中明文"需避免的写法"？
- [ ] `voice.md`中"需积极采用的写法"是否有所体现？

## 结构与逻辑
- [ ] 章节顺序是否符合`structure.md`的框架？
- [ ] 同级标题之间是否存在逻辑并列或递进关系（而非混乱堆砌）？
- [ ] 段落之间过渡是否自然？
- [ ] 是否有明确的引言、正文和结论结构？
- [ ] 是否形成了完整的论证链条？

## 整体可读性
- [ ] 标题是否准确概括了对应内容？
- [ ] 专业术语的使用是否前后一致？
- [ ] 全文是否存在明显的语病或错别字？
- [ ] 字数是否在3000-3500字范围内？

## 自查流程建议
1. 先通读全文，对照`topic.md`核验主题聚焦度。
2. 逐章对照`structure.md`，勾选内容要点。
3. 抽查3-5段，对照`voice.md`核验风格。
4. 检查是否有足够的个人经历与书中内容的结合。
5. 完成清单勾选，记录发现的问题。

---

**使用说明**：此文件供你（或AI）在文档生成后进行复核。你可以直接在方框内打勾，或在发现问题时注明具体段落。
